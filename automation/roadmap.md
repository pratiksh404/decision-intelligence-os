# Automation Roadmap

## Objective

Turn the markdown operating system into an increasingly automated intelligence platform while preserving founder control over final decisions.

## Phase 1: Structured Markdown Discipline

- Enforce templates.
- Standardize metadata.
- Link artifacts across decisions, forecasts, opportunities, and sources.
- Maintain review dates.

## Phase 2: Source Ingestion

- RSS feeds
- Newsletters
- Saved links
- Company blogs
- Filings
- Odds markets
- Polymarket pages
- Sports data sources
- Financial market data

## Phase 3: Classification

Classify incoming signals as:

- Opportunity
- Threat
- Competitor Signal
- Market Trend
- Forecast Update
- Execution Risk
- Source Note

## Phase 4: Draft Generation

Generate draft artifacts:

- Intelligence reports
- Opportunity memos
- Forecast updates
- Weekly CEO reviews
- Decision memo drafts

## Phase 5: Agent Orchestration

Run advisory-board reviews automatically:

1. Intelligence Analyst summarizes evidence.
2. CEO, CFO, VC, Skeptic, and Operator critique independently.
3. Probability Analyst produces scenarios and probabilities.
4. System generates final recommendation.

## Phase 6: Dashboards

Build dashboards for:

- Active decisions
- Open opportunities
- Forecast ledger
- Execution plans
- Review dates
- Source quality
- Confidence distribution

## Phase 7: Calibration and Learning

- Track forecast accuracy.
- Compare decisions against outcomes.
- Identify repeated judgment errors.
- Update templates and workflows.
