# Forecasting Framework

## Purpose

Forecast uncertain outcomes using explicit probabilities, evidence, counter-evidence, and calibration.

## Required Forecast Structure

- Question
- Time horizon
- Resolution criteria
- Evidence
- Counter-evidence
- Unknown variables
- Scenario analysis
- Probability estimates
- Confidence rating
- Review schedule

## Probability Discipline

Do not produce binary predictions. Use probability ranges when uncertainty is high.

Example:

- Best case: 25%
- Base case: 50%
- Worst case: 25%

## Forecast Quality Standard

A forecast is useful when:

- It can resolve true or false by a defined date.
- It states what would change the probability.
- It identifies the most important unknowns.
- It includes counter-evidence.
- It can be reviewed for calibration.

## Calibration Loop

After resolution:

1. Record actual outcome.
2. Compare outcome to probability estimate.
3. Identify bias or missed signal.
4. Update future base rates.
5. Link the resolved forecast to related decisions.
