# Execution Planning Workflow

## Purpose

Convert approved decisions and opportunities into concrete action plans with KPIs, risks, and review cadence.

## Required Inputs

- Approved decision or opportunity
- Objective
- Constraints
- Time horizon
- Available resources
- Success criteria

## Required Plan

- Immediate Actions
- 7-Day Plan
- 30-Day Plan
- 90-Day Plan
- KPIs
- Risks
- Success Metrics
- Review Schedule

## Execution Principles

- Define the smallest useful next action.
- Use measurable milestones.
- Separate activity metrics from outcome metrics.
- Identify dependencies early.
- Review progress weekly.

## Output Location

Store execution plans in `vault/05-execution/`.
