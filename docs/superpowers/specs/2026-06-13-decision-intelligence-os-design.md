# Decision Intelligence OS Design

## Objective

Create a production-grade markdown repository named `decision-intelligence-os` that functions as a hybrid Obsidian intelligence vault and portable AI workflow system for a founder/CEO.

## Design Decision

Use a hybrid architecture:

- Obsidian vault for durable artifacts, metadata, links, retrieval, and reviews.
- Agent/workflow repository for portable AI operating protocols across multiple coding and assistant tools.

## Required Components

- README with vision, architecture, folder structure, agents, workflows, usage, reviews, forecasting, opportunity process, and automation roadmap.
- Advisor definitions for CEO, CFO, VC, Skeptic, Operator, Intelligence Analyst, and Probability Analyst.
- Workflows for opportunity analysis, market intelligence, forecasting, decision memo, execution planning, and weekly CEO review.
- Templates for opportunity, forecast, decision memo, intelligence report, and weekly review.
- Operating-system documents for principles, decision framework, forecasting framework, confidence framework, metadata schema, and review cadence.
- Obsidian-ready vault structure.

## Quality Standard

Every created markdown file must contain professional content. No empty placeholder files should be created.

## Verification

Verify that required files exist, required sections are present, and no obvious placeholder language remains.
